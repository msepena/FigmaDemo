## Project

SwiftUI iOS app: a tic-tac-toe game with a Settings screen. Display name "Tic Tac Toe", bundle id `com.test.TicTacToeGame`, universal (iPhone + iPad), Swift 5.0 (app target) / Swift 6.0 tools (package), **iOS deployment target 26.4** (set deliberately — do not lower it without asking). The Xcode project, schemes, folders, and modules are still named `FigmaDemo` / `FigmaDemoKit`; only the product's display name and bundle id were renamed.

The AppIcon (`Icon-Light/Dark/Tinted.png`) and the static `LaunchLogo` PNGs are generated by `Tools/AssetGen` (`swift run --package-path Tools/AssetGen AssetGen`). Don't hand-edit the PNGs — change the renderer in `Tools/AssetGen/Sources/AssetGen/IconRenderer.swift` and rerun.

The app entry point is `FigmaDemoApp` → `ContentView`, which owns an `AppPreferences` and a `GameViewModel` and drives a `NavigationStack` between `GameScreen` and `SettingsScreen` (both come from `FigmaDemoKit`).

## Architecture

The repo is an `.xcodeproj` app target that links a sibling **local Swift package** (`FigmaDemoKit/Package.swift`, swift-tools 6.0) containing the bulk of the code. The package is referenced by the Xcode project as an `XCLocalSwiftPackageReference`; the app target depends on `AppPreferences`, `DesignSystem`, `GameFeature`, and `SettingsFeature`.

Module graph (lowest → highest):

- **AppPreferences** — `@Observable` wrapper over `UserDefaults` exposing typed prefs: `Theme`, `AccentColor`, `MarkerStyle`, `FirstMove`, `Difficulty`.
- **GameDomain** — Pure model/logic, no SwiftUI: `Player`, `Mark`, `Board`, `CellPosition`, `WinningLine`, `GameState`, `GameOutcome`, `GameEngine`, plus the `AIOpponent` protocol with `RandomAI` / `BlockOrWinAI` / `ForkAwareAI` strategies.
- **DesignSystem** — Token primitives: `DSColor`, `DSFont`, `DSRadius`, `DSShadow`, `DSSpacing`, plus `Theming` (`dsAccentColor` environment, etc.).
- **UIComponents** (depends on DesignSystem) — Reusable SwiftUI views: `BoardCellView`, `BoardCardContainer`, `ScoreboardCard`, `TurnIndicatorPill`, `SettingsCard`, `SettingsRow`, `SegmentedPicker`, `PrimaryPillButton`, `CircularIconButton`, `DisclosureValueRow`, `DestructiveTextRow`, `AccentSwatchPicker`, `EyebrowText`, `ScreenHeader`, `TextLinkButton`.
- **GameFeature** (depends on AppPreferences, GameDomain, DesignSystem, UIComponents) — `GameScreen`, `GameViewModel` (`@Observable @MainActor`), `GameScreen+Mappings`, `FirstMove+Player`, `AIOpponentFactory`. The view model owns the AI move loop: it determines `humanSide` from `prefs.firstMove`, picks an `AIOpponent` via `AIOpponentFactory.make(for: prefs.difficulty)`, and schedules the AI via a cancellable `Task` with a configurable `aiMoveDelay`.
- **SettingsFeature** (depends on AppPreferences, DesignSystem, UIComponents) — `SettingsScreen`, `SettingsViewModel`, `SettingsScreen+Mappings`. The view model accepts an `onResetStats` callback so the app target can wire it to `GameViewModel.resetAll()`.

## Build & test

The project uses an `.xcodeproj` plus a local SwiftPM package (`FigmaDemoKit`). Open `FigmaDemo.xcodeproj` in Xcode, or use `xcodebuild` from a shell with full Xcode selected (`sudo xcode-select -s /Applications/Xcode.app`). The bare Command Line Tools install does not include `xcodebuild`.

Common commands:

```bash
# Build the app for simulator
xcodebuild -project FigmaDemo.xcodeproj -scheme FigmaDemo \
  -destination 'platform=iOS Simulator,name=iPhone 16' build

# Run all app-target tests (FigmaDemoTests + FigmaDemoUITests)
xcodebuild -project FigmaDemo.xcodeproj -scheme FigmaDemo \
  -destination 'platform=iOS Simulator,name=iPhone 16' test

# Run the package's tests via the FigmaDemoKit scheme (auto-generated by SPM)
xcodebuild -scheme FigmaDemoKit \
  -destination 'platform=iOS Simulator,name=iPhone 16' test

# Run a single Swift Testing test (Target/Suite/test)
xcodebuild -scheme FigmaDemoKit \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  -only-testing:GameFeatureTests/GameViewModel/freshViewModelStartsAtRound1 test

# Run a single XCTest (Target/Class/method)
xcodebuild -project FigmaDemo.xcodeproj -scheme FigmaDemo \
  -destination 'platform=iOS Simulator,name=iPhone 16' \
  -only-testing:FigmaDemoUITests/FigmaDemoUITests/testExample test
```

## Test layout

Each library target has a parallel test target under `FigmaDemoKit/Tests/`:

- `AppPreferencesTests`, `GameDomainTests`, `DesignSystemTests`, `UIComponentsTests`, `GameFeatureTests`, `SettingsFeatureTests` — all **Swift Testing** (`import Testing`, `@Test`, `#expect`, `@Suite`).
- `UIComponentsTests` additionally depends on [`pointfreeco/swift-snapshot-testing`](https://github.com/pointfreeco/swift-snapshot-testing) (`from: 1.17.0`).

App-target tests live at the repo root:

- `FigmaDemoTests/` — **Swift Testing**.
- `FigmaDemoUITests/` — **XCTest** (`XCTestCase`, `XCUIApplication`). Required because Swift Testing does not cover UI automation.

Don't mix the two within a single target. New tests in any package target follow Swift Testing conventions.

## Linting

SwiftLint config lives at `.swiftlint.yml`. A few defaults are turned off deliberately (`trailing_comma`, `colon`, `comma`, `opening_brace`, `large_tuple`, `static_over_final_class`) and `identifier_name.min_length` is relaxed to 1 to allow domain-meaningful short names (`Player.x`/`.o`, `r`/`c`, design-token tokens like `xs`/`sm`/`md`, etc.). When adding code, prefer to fit the existing style rather than re-enabling these rules.
